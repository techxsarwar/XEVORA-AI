## ⚡ What is Xevora AI?

> **Xevora AI Vibe Studio** is a next-generation, developer-first AI coding environment. It routes intelligently across multiple LLMs, generates clean modular code, and lets you preview it live — all without leaving your workflow.

No copy-paste. No downtime. No spaghetti. Just signal.

<img src="https://user-images.githubusercontent.com/73097560/115834477-dbab4500-a447-11eb-908a-139a6edaec5c.gif" width="100%">

## 🚀 Features

<table>
<tr>
<td width="50%">

### 🧠 Multi-Model Matrix Routing
Xevora employs an intelligent fallback matrix. If your primary model hits rate limits or goes dark, Xevora auto-routes to secondary heavy-hitters like **Llama 3 70B** and **Qwen Coder** — keeping your workflow at full throttle with **zero downtime**.

</td>
<td width="50%">

### 📦 Modular Multi-File Generation
The engine forces the AI to decouple output into clean, independent modules — `index.html`, `styles.css`, `app.js`. An industrial-grade markdown parser auto-extracts, tags, and organizes every file. No more monolith blobs.

</td>
</tr>
<tr>
<td width="50%">

### ⚡ Instant Live Preview Sandbox
Hit **Live Preview** on any AI response. The assembly engine stitches raw code blocks in milliseconds and spawns a full sandboxed DOM execution environment right inside your chat. Real code. Real results. Instantly.

</td>
<td width="50%">

### 🎨 Terminal-Dark Design System
A sleek, premium, terminal-dark substrate UI — monospace-first, zero bloat. Built with developer ergonomics at the core. The interface gets out of your way and lets the code breathe.

</td>
</tr>
</table>

## ⚙️ Quick Start

```bash
# 01 — Clone the repository
git clone https://github.com/techxsarwar/XEVORA-AI.git
cd XEVORA-AI

# 02 — Install dependencies
npm install

# 03 — Configure your API key
echo "VITE_OPENROUTER_API_KEY=your_api_key_here" > .env.local

# 04 — Fire it up
npm run dev
```

> 🔑 Get your OpenRouter API key at [openrouter.ai](https://openrouter.ai)

```

├── 📁 src/
│   ├── 📄 main.ts           # App entry point
│   ├── 📄 router.ts         # Multi-model fallback matrix
```
